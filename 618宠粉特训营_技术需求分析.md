# 618宠粉特训营 HTML 页面 — 技术需求分析文档

> 版本：v1.0 | 日期：2026-06-14 | 基于「百日练兵计划设计方案」+「618宠粉特训营计划」

---

## 一、项目背景

弘历软件官方服务频道于 2026 年 6 月正式启动**百日练兵计划**——一个定位为长期运营的会员成长项目。该计划以「易学久会」为基础训练场、「财经壹眼」为进阶成长营，通过里程碑解锁机制持续驱动会员参与。

618 作为百日练兵计划启动后的**首场成长战役**，频道推出 **「百日炼兵 · 618 宠粉特训营」**。活动时间为 **6 月 13 日 — 6 月 29 日**，核心商业目标是销售弘历软件终免信息费（原价 ¥14,800 → 折扣价 ¥14,182，立减 ¥618），转化目标 50 人。

本次活动需要一个 **HTML 活动页面** 作为线上承载主体，用于展示活动信息、引导用户参与成长挑战、展示领航员权益，并最终导向特训营报名/转化。

## 五、交互需求

### 5.1 核心交互

| 交互项 | 触发条件 | 行为 | 优先级 |
|--------|----------|------|--------|
| 倒计时 | 页面加载 | 实时倒计时至 6/29 23:59:59 | P0 |
| 日期高亮 | 页面加载 | JS 判断当前日期，高亮对应 Day 卡片 | P0 |
| 已完成标记 | 页面加载 | 当前日期之前的 Day 显示「已完成」勾 | P0 |
| 滚动导航 | 滚动/点击导航点 | 切换当前屏，导航点联动 | P0 |
| 报名/领取按钮 | 点击 | 弹出表单（精网号输入）或跳转外部链接 | P0 |
| 提交成功反馈 | 表单提交 | 弹窗提示 + 礼花特效 + 滚动条更新 | P1 |
| 权益卡片 hover | 鼠标悬停 | 上浮+发光效果 | P1 |
| 里程碑路径动画 | 进入视口 | 逐级点亮里程碑节点 | P2 |

### 5.2 日期驱动逻辑（关键）

```javascript
// 伪代码
const today = new Date(); // 2026-06-14
const daySlots = [
  { day: 1, date: '2026-06-15', label: 'Day1' },
  { day: 2, date: '2026-06-16', label: 'Day2' },
  { day: 3, date: '2026-06-17', label: 'Day3' },
  { day: 4, date: '2026-06-18', label: 'Day4' },
  { day: 5, date: '2026-06-19', label: 'Day5' },
];

// today < date   → 锁定态（灰色）
// today = date   → 进行中（高亮发光）
// today > date   → 已完成（绿色勾）
// today > 6/29   → 活动结束
```

### 5.3 移动端适配

- 需覆盖 360px — 2560px 全分辨率
- 全屏滚动在移动端使用 `100dvh` 兜底
- 时间线卡片在小屏改为纵向排列
- 权益卡片在平板以下改为 2 列或单列

## 七、数据流向

```
┌──────────┐    ┌──────────┐    ┌──────────────┐
│ 用户浏览器 │ →  │ HTML页面  │ →  │ 飞书多维表格   │
│ 输入精网号 │    │ 表单提交  │    │ (数据收集)     │
└──────────┘    └──────────┘    └──────────────┘
                       │
                       ▼
                ┌──────────┐
                │ 本地计数  │  (已参与人数、滚动记录)
                │ + 礼花反馈│
                └──────────┘
```

- **表单收集**：复用已有飞书表单方案，或嵌入飞书 iframe
- **计数展示**：页面本地维护计数器（初始值可预设），每次提交 +1
- **滚动记录**：预设 40 条模拟记录 + 用户提交后即时追加

## 八、技术选型

| 技术点 | 方案 | 理由 |
|--------|------|------|
| 页面结构 | 纯静态 HTML 单文件 | 托管 GitHub Pages，频道以超链接嵌入 |
| 样式 | 原生 CSS（无框架） | 单文件自包含，加载快，无依赖 |
| 滚动 | CSS `scroll-snap` | 原生支持好，无需 JS 库 |
| 动画 | CSS Animation + Canvas | 星空/光晕用 CSS，礼花/K线用 Canvas |
| 日期逻辑 | 原生 JS Date | 纯前端计算，无后端依赖 |
| 表单 | 飞书 iframe 嵌入 或 自建表单 | 自建表单 + 提醒用户手动录入飞书 |
| 托管 | GitHub Pages | 用户既有方案，链接稳定 |

## 十、风险与注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 日期驱动逻辑错误 | 用户看到错误的 Day 高亮 | 充分测试各日期边界（6/14、6/15、6/18、6/19、6/29、6/30） |
| 价格信息变更 | 页面展示与实际不一致 | 价格数字集中定义为 JS 常量，方便修改 |
| 表单数据未自动入库 | 收集的精网号需手动录入 | 页面明确提示「客服将在24小时内联系」，给缓冲期 |
| 移动端显示异常 | 部分用户无法正常浏览 | 覆盖 360px-2560px，使用 clamp() 流体尺寸 |
| GitHub Pages 更新延迟 | 上线后修改不即时生效 | 文件名加版本号，或使用 cache-busting 参数 |

> 版本：v2.0 | 2026-06-14 | 基于百日练兵计划 + 618宠粉特训营方案
> 重大更新：拆分为领航员版 + 半年版用户版双版本；特训营开营前=宣传期，页面终点=领取优惠券

## 一、设计策略

### 1.1 双版本定位

本次页面面向两类用户，设计上需要一份代码输出两个版本：

| 版本 | 目标用户 | 用户状态 | 核心路径 |
|------|----------|----------|----------|
| **半年版用户版** | 半年版网员（291人在网） | 非领航员，需转化 | 宣传特训营 → 成长挑战 → 获取入场券 → 进入特训营 → 领取优惠券 |
| **领航员版** | 已是终身版领航员 | 已有身份，无需入场券 | 特训营开幕 → 直接领取优惠券 |

**版本切换机制**：页面顶部提供版本标签切换入口，或通过 URL 参数 `?v=half` / `?v=leader` 区分，便于运营分别投放链接。

### 1.2 时间线与页面边界

```
6/13 ───────────── 6/18 ─── 6/19 ─── 6/21 ───── 6/29
│←── 宣传期（开营前）──→│←─ 特训营期间 ─→│←─ 全活动期 ─→│
│                        │               │
│  HTML 展示：宣传特训营   │  HTML 展示：    │
│  不涉及优惠券           │  特训营+优惠券  │
│                        │               │
│  半年版：获取入场券      │  半年版：凭入场券进入特训营 → 解锁优惠券
│  领航员：无需入场券      │  领航员：直接领取优惠券
│                        │               │
└── HTML 页面终点 = 领取优惠券 ──────────┘
```

**关键约束**：无论哪个版本，HTML 页面的终点都是「领取优惠券」。页面不承载特训营内部课程内容。

### 1.3 设计关键词

**宣传造势（开营前） → 身份区隔（双版本） → 优惠券终点**

## 三、版本区隔机制

### 3.1 版本切换入口

页面顶部固定栏（sticky，z-index 最高），包含两个版本标签：

```
┌──────────────────────────────────────────────┐
│  [半年版用户]  [领航员]         618宠粉特训营  │
│   ◄ 当前激活 ►                               │
└──────────────────────────────────────────────┘
```

- 半年版标签：金边透明底，激活时填充金色
- 领航员标签：蓝色调，激活时填充版本蓝
- 点击切换时，页面内容平滑过渡（fade 0.3s）
- URL 参数同步更新（`?v=half` / `?v=leader`）

### 3.2 各版本独有的视觉元素

| 元素 | 半年版用户版 | 领航员版 |
|------|-------------|----------|
| 顶部标签色 | 金色 | 版本蓝 `#4a90d9` |
| 角标文字 | 「限时立减 ¥618」 | 「领航员专属」 |
| 第二屏 | Day1-5 成长挑战时间线 | 跳过，直接进入权益 |
| 入场券 | 有（Day4 奖励） | 无（无需入场券） |

## 六、动画体系

### 6.1 全局氛围动画

| 动画名称 | 实现方式 | 参数 |
|----------|----------|------|
| 星空闪烁 | CSS `@keyframes twinkle` | 80颗星星，2-5s周期，20%金色 |

| K线图 | Canvas 绘制 | 红绿蜡烛图，5组不同参数 |
| 版本切换过渡 | CSS `transition` opacity | 0.3s fade |

### 6.2 组件交互动画

| 触发条件 | 动画 | 时长 |
|----------|------|------|
| 版本标签切换 | 内容淡入淡出 | 0.3s |
| 导航点激活 | 背景色填充 + 外发光 | 0.3s |
| 按钮 hover | 上移 3px + 阴影增强 | 0.35s |
| 按钮 active | 回弹原位 | 0.15s |
| 权益卡片 hover | 上移 4-6px + 边框发光 | 0.3s |
| 时间线卡片（今日） | 边框脉冲（opacity 0.2→0.5） | 2s 循环 |
| 优惠券卡片 hover | 微光扫描效果 | 3s 循环 |
| 弹窗出现 | `popIn` scale+translate | 0.4s |
| 弹窗对号 | 脉冲光环扩散 | 1.5s 循环 |
| 礼花 | Canvas 粒子物理 | ~2s 消散 |
| 提交记录滚动 | CSS `marquee` 循环 | 20s 线性 |

### 6.3 动画禁用策略

- 所有动画使用 CSS 实现，不依赖 JS 定时器（除 Canvas 礼花/K线）
- 移动端减少粒子数量：礼花从 60→30 粒子
- `prefers-reduced-motion` 媒体查询预留（可选）

## 七、响应式断点设计

### 7.1 断点规划

| 断点 | 宽度范围 | 设计策略 |
|------|----------|----------|
| XS | ≤400px | 最小手机，字号大幅缩小，版本标签紧凑排列 |
| SM | 401-768px | 手机横屏/平板竖屏，时间线卡片全宽 |
| MD | 769-1100px | 平板横屏/小桌面，标准布局 |
| LG | 1101-1920px | 桌面标准，内容区 max-width 1100px 居中 |
| XL | ≥1920px | 大屏，max-width 扩至 1300px，字号上浮 |

### 7.2 移动端特殊处理

| 组件 | 桌面 | 手机 |
|------|------|------|
| 版本标签 | 并排 120px 宽 | 并排 70px 宽，字号缩小 |
| 导航点 | 右侧 24px | 右侧 8px，缩小至 8px |
| 时间线卡片 | 固定宽度横向排列 | 全宽纵向堆叠 |
| 权益矩阵 | 2行×3列 | 3行×2列（≤400px 改为单列） |
| 双栏目卡片 | 并排 | 上下堆叠 |
| 优惠券卡片 | 最大 420px 宽 | 全宽 90vw |
| 倒计时 | gap 16px | gap 8px，数字缩小 |
| K线装饰 | 可见 | 隐藏（减少性能开销） |

### 7.3 字体流体方案

所有尺寸使用 `clamp(min, vw/vh_ratio, max)`：

```css
font-size: clamp(11px, 1.2vw, 13px);  /* 正文 */
font-size: clamp(24px, 3.5vw, 36px);  /* 倒计时数字 */
padding: clamp(14px, 1.8vh, 18px) clamp(48px, 8vw, 80px); /* 按钮 */
```
